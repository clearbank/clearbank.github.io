title: "SEPA Instant Recall Request Received webhook"
version: 2
webhook: true
---

This webhook confirms an outgoing SEPA recall request has been received.

#### Request body

```json
{
    "Type":"Sepa.Instant.RecallRequest.Received",
    "Version":2,
    "Payload": {...},
    "Nonce":
}
```

#### Payload
| Element                 | Description     |
|-------------------------|-----------------|
|`OriginalTransactionId` |**Required**<br />Unique identifier of the original payment transaction, as assigned by ClearBank, to unambiguously identify the transaction. This identifier is passed on, unchanged, throughout the entire end-to-end chain.<br/>Type: `string`<br/>Max. Length: `36`<br/>Min. Length: `1` |
|`ReasonCode` |**Required**<br />The reason code used to recall the payment. This can be DUPL, FRAD, TECH, CUST, AC03 or AM09.<br/>Type: `string`|
|`TimestampCreated` |**Required**<br />Date and time the recall was requested, in UTC.<br/>Type: `string`<br/>Format: `date-time`|
|`AdditionalInformation` |Additional information provided by the initiating party about the recall.<br/>Type: `array` or `null`<br />Type: `string`|
|`PaymentId` |**Required**<br />Unique identifier for payment as assigned by ClearBank.<br/>Type: `string` <br/>Format: `uuid`|
