title: "SEPA Instant Recall Request Rejected webhook"
version: 2
webhook: true
---

This webhook confirms an outgoing SEPA Instant recall request has been rejected.

#### Request body

```json
{
    "Type":"Sepa.Instant.RecallRequest.Rejected",
    "Version":2,
    "Payload": {...},
    "Nonce":
}
```

#### Payload
| Element                 | Description     |
|-------------------------|-----------------|
|`OriginalTransactionId` |**Required**<br />Unique identifier of the original payment transaction, as assigned by ClearBank, to unambiguously identify the transaction. This identifier is passed on, unchanged, throughout the entire end-to-end chain.<br/>Type: `string`<br/>Max. Length: `36`<br/>Min. Length: `1` |
|`RejectionCode` |**Required**<br />Specifies the reason for the rejection of the recall. External code sets can be downloaded from [www.iso20022.org](www.iso20022.org).`NOOR`-Original credit transfer never received. `RNPR`-Original credit transfer was received but is not processable. `ARJT`-Already rejected original SCT (SEPA Credit Transfer). `ARDT`-Already returned original SCT (SEPA Credit Transfer). `RR04`-Regulatory reason.<br/>Type: `string`|
|`TimestampRejected` |**Required**<br />Date and time the recall was rejected, in UTC.<br/>Type: `string`<br/>Format: `date-time`|
|`AdditionalInformation` |Additional information provided by the initiating party about the rejection.<br/>Type: `array` or `null`<br />Type: `string`|
|`PaymentId` |**Required**<br />Unique identifier for payment as assigned by ClearBank.<br/>Type: `string` <br/>Format: `uuid`|
