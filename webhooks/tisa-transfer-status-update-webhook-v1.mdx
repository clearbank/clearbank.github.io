title: "Tisa Transfer Status Update webhook"
version: 1
webhook: true
---

This webhook notifies that the status of an automated cash ISA transfer has been updated.



#### Request body

```json
{
    "Type":"TisaTransferStatusUpdate",
    "Version":1,
    "Payload": {...},
    "Nonce":
}
```


#### Payload
| Element                   | Description  |
|---------------------------|--------------|
| `MessageId`               | **Required**<br /> The unique message reference from the Swift message. <br /> Type: `string` <br /> Min length: `1` <br /> Max length: `35` <br /> Pattern: `^[A-Z0-9\\-]{35}$` |
| `TimestampCreated`               | **Required**<br />Timestamp created. <br /> Type: `string` <br /> Min length: `1` <br /> Format: `date-time` |
| `ReceiverSun`               | **Required**<br />The SUN (Service User Number) of the party receiving the status. <br /> Type: `string` <br /> Min length: `1` <br /> Max length: `6` <br /> Pattern: `^[0-9]{6}$` |
| `InitiatorSun`               | **Required**<br />The SUN (Service User Number) of the party issuing the status. <br /> Type: `string` <br /> Min length: `1` <br /> Max length: `6` <br /> Pattern: `^[0-9]{6}$` |
| `Reference`               | **Required**<br />This will normally be set to NA. If TECHREJECT, the ID of the rejected message will be inserted into this field. <br /> Type: `string` <br /> Min length: `1` |
| `Urn`               | **Required**<br />Unique Reference Number(URN). <br /> Type: `string` <br /> Min length: `1` <br /> Max length: `18` <br /> Pattern: `^[A-Z0-9]{18}$` |
| `State`               | **Required**<br />The updated state of the transfer. Valid options include: `Default`, `Completed`, `Abandoned`, `Pending`, `Referred`, `Rejected`, `ReadyForPayment`, `Failure`, `TechnicalRejection`, `PaymentReceived`, `InvalidPaymentReceived` <br /> Type: `string` |
