## Savings accounts overview

A savings account is an FSCS-protected bank account that can accrue interest on behalf of a retail or business customer. Savings accounts may be held individually or jointly, and you can update their interest rate configuration at will.

ClearBank calculates interest accruals on savings account deposits based on pre-agreed interest configurations. These accruals are posted for you as regular interest payments to your customers' savings accounts. To learn more about interest configurations, refer to [Interest configuration](./interest-configuration).

If you're interested in using savings accounts, please contact your Client Director.

<Callout colour="blue">
  <strong>Note:</strong> Deposits held in savings accounts may be eligible for FSCS protection. FSCS protection insures deposits of up to £120,000 per person or legal entity per authorised firm (ClearBank) in the event of insolvency. Customers should review the specific terms and conditions of their account to confirm whether FSCS protection applies.
</Callout>

## Create a savings account

Savings accounts have several interdependencies. When creating a savings account, follow the below steps to ensure you cover them all:

1. Check you've created the **retail** or **business customer** this new savings account will belong to. To create a retail customer, refer to the [Create a retail customer](./retail-customers#create-a-retail-customer) endpoint.
2. Confirm which **interest configuration** should apply to the account (`interestConfigurationId`).  
  Interest configuration IDs refer to a specific interest accrual arrangement, and are provided to you directly by a ClearBank contact. Refer to [Interest configuration](./interest-configuration) for more details.  
3. Create the **savings account** by calling the [POST /v2/savings](#create-a-savings-account-endpoint) endpoint, using the customer(s) and the interest configuration's IDs.
4. Specify a **nominated account** to fund or defund the savings account using the [PUT /v1/accounts/{accountId}/nominated-account](#set-a-nominated-account) endpoint.  
  **Note:** A <strong>nominated account</strong> is the account that accrued interest, if any, will be transferred into upon closure of the savings account. Name matching between savings and nominated accounts is your responsibility.

![Submit the POST /v2/savings endpoint. Once you receive a 201 response, submit the PUT /v1/accounts/{accountId}/nominated-account to set up a nominated account. Once you receive a 200 response, funds can now be added to the savings account.](/assets/images/savings-account-open.svg)

<Callout colour="blue">
Always take note of the X-Request-Id when requesting creation of an account. In the event you receive a 500 response, you can resubmit your request using the same X-Request-Id to prevent accidental duplication of the account.
</Callout>

Once you've opened a savings account, you can deposit and withdraw payments via Faster Payments or CHAPS. Savings accounts cannot send or receive Bacs payments.

### Creating joint savings accounts

Both single and joint savings accounts can be created using the [POST /v2/savings](#create-a-savings-account-endpoint) endpoint. To make joint accounts using this endpoint, specify each customer's ID in the `customers` array.

#### Example payload: two customers

```json
{
  "ownerName": "John and Jane Smith",
  "sortCode": "010203",
  "productId": "1a2b3c4d-5e6f-7a8b-9c0d-1e2f3a4b5c6d",
  "customers": [
    {
      "customerId": "0f1e2d3c-4b5a-6c7d-8e9f-0a1b2c3d4e5f",
      "customerId": "abcdef12-3456-7890-1234-56789abcdef0"
    }
  ],
  "interestConfigurationId": "1234abcd-5678-9ef0-1234-56789abcdef0"
}
```

#### Example payload: four customers

While joint accounts will typically have two customers, we can support any number of customers where there is a valid use case.

```json
{
  "ownerName": "John Smith, Jane Smith, Steve Doe, and Emily Doe",
  "sortCode": "010203",
  "productId": "a1b2c3d4-e5f6-a7b8-c9d0-e1f2a3b4c5d6",
  "customers": [
    {
      "customerId": "9f8e7d6c-5b4a-3c2d-1e0f-9a8b7c6d5e4f",
      "customerId": "0a1b2c3d-4e5f-6a7b-8c9d-0e1f2a3b4c5d",
      "customerId": "f1e2d3c4-b5a6-c7d8-e9f0-a1b2c3d4e5f6",
      "customerId": "2b3c4d5e-6f7a-8b9c-0d1e-2f3a4b5c6d7e"
    }
  ],
  "interestConfigurationId": "4d5e6f7a-8b9c-0d1e-2f3a-4b5c6d7e8f9a"
}
```

## Close a savings account

**To close a savings account:**
1. Withdraw all funds from the account.
2. Verify that a nominated account has been defined. You can obtain details of the currently configured nominated account using the [GET /v1/accounts/{accountId}/nominated-account](#get-a-nominated-account) endpoint.
3. Use the [POST /v1/accounts/{id}/closure](#close-a-savings-account) endpoint.
> The process is asynchronous and can take longer if an interest calculation is pending. Unpaid accrued interest, if any, will be deposited into the nominated account. If there is no interest to be paid, then it's possible to close the account without a nominated account being defined.
4. OPTIONAL Check closure status using the [GET /v1/accounts/{id}/closure](#get-account-closure-status) endpoint.
5. The [Account Closure Completed webhook](#account-closure-completed-webhook) will be sent once the account has closed.

![To close an account, set the nominated account using the PUT /v1/accounts/{accountId}/nominated-account endpoint. Once you receive a 200 response, submit the PUT /v1/accounts/{accountId}/nominated-account to set up a nominated account. Once you receive a 200 response, funds can now be added to the savings account.](/assets/images/savings-account-close.svg)

<Callout colour="blue">
Webhooks are not instant and may take up to the following day to arrive. Wait until you receive a webhook before reattempting account closure.
</Callout>

Account closure may fail due to the nominated account details being incorrect. You will know from the `ClosureFailureReason` field in the Account Closure Failed webhook.

See the following possible combinations of `accountStatus` and `closureStatus`:

| accountStatus | closureStatus | Description                                                            |
| ------------- | ------------- | ---------------------------------------------------------------------- |
| Enabled       | None          | No closure request submitted.                                          |
| Enabled       | Pending       | Closure requested, interest being calculated.                          |
| Enabled       | Failed        | Closure request failed, see `closureFailureReasons` field for details. |
| Suspended     | Pending       | Closure requested for suspended account, interest being calculated.    |
| Closed        | Complete      | Closure completed successfully.                                        |

<br/> <br/>
<Callout colour="blue">
If account closure fails and hasn't been resolved by the end of the month, then accrued interest, if any, will be paid into the account. You will need to withdraw these funds from the account again before reattempting closure.
</Callout>

**To reset the nominated account and close the account:**
1. OPTIONAL Obtain details of the currently configured nominated account using the [GET /v1/accounts/{accountId}/nominated-account](#get-a-nominated-account) endpoint.
2. Correct the nominated account details using the [PUT /v1/accounts/{accountId}/nominated-account](#set-a-nominated-account) endpoint.
4. Use the [POST /v1/accounts/{id}/closure](#close-a-savings-account) endpoint.
> The process is asynchronous. Unpaid accrued interest, if any, will be deposited into the nominated account.
5. OPTIONAL Check closure status using the [GET /v1/accounts/{id}/closure](#get-account-closure-status) endpoint.
6. The [Account Closure Completed webhook](#account-closure-completed-webhook) will be sent once the account has closed.

Once you receive a 200 response, submit the PUT /v1/accounts/{accountId}/nominated-account to set up a nominated account. Once you receive a 200 response, funds can now be added to the savings account.](/assets/images/savings-account-close-fail-nominated-account.svg)
